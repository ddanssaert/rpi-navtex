   ```python
   def _matches_filters(sub_filters, station_id: str, message_type: str) -> bool:
       """Return True if the subscription should receive a message with these
       station/type values. NULL filters or empty lists mean 'send all'."""
       if sub_filters is None:
           return True
       stations = sub_filters.get("stations", [])
       types = sub_filters.get("types", [])
       return (len(stations) == 0 or station_id in stations) and \
              (len(types) == 0 or message_type in types)
   ```

   ```python
   @app.put("/push/filters")
   async def update_push_filters(update: PushFiltersUpdate, db: AsyncSession = Depends(get_db)):
       from sqlalchemy.future import select
       from fastapi import HTTPException
       result = await db.execute(
           select(PushSubscription).where(PushSubscription.endpoint == update.endpoint)

       if not sub:
   ```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | PUT /push/filters | mitigate | Endpoint-as-identity: only the holder of the push endpoint URL can target a row. Pydantic `PushFiltersUpdate` validates shape before DB write. 404 on unknown endpoint prevents creation by spoofing. |
| T-10-02 | Information Disclosure | PUT /push/filters | accept | No PII in filters (single-char station/type IDs). Endpoint URLs are already considered public-ish secrets in WebPush threat model. |
| T-10-03 | DoS | PUT /push/filters | accept | LAN-only deployment, no auth boundary. Rate-limiting deferred to infra layer. |
| T-10-04 | Tampering | _matches_filters | mitigate | Helper treats NULL/missing keys defensively (`.get("stations", [])`); cannot raise on malformed JSON in DB. |
| T-10-05 | Repudiation | POST /push/subscribe filters field | accept | No audit log; subscription registration is anonymous by design. |
</threat_model>
