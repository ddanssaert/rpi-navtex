<objective>
Catch iOS silent push-subscription expiry by re-registering automatically on app open.

Purpose: iOS WebKit silently invalidates push subscriptions over time (across iOS updates, prolonged inactivity, or storage pressure). The user previously granted permission and subscribed, but the next push fails server-side with HTTP 410 because the subscription is no longer valid on the device. Per D-08, on every PWA open we check whether a subscription exists and silently re-register if it does not — but only when permission is still `'granted'` (otherwise we'd be re-subscribing without user consent, which is impossible anyway).

Output: One small extension to the existing subscription-check `useEffect` in `frontend/src/components/Settings.jsx`.
</objective>
